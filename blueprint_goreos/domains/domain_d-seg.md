# D-SEG: Seguridad Pública Regional

> **Dominio:** Seguridad Pública Regional  
> **Código:** D-SEG  
> **Capa:** Núcleo Táctico  
> **Función:** PROTEGER  
> **División Responsable:** División de Prevención del Delito  
> **Versión:** 1.0.0  
> **Fecha:** Diciembre 2024

---

## 1. Propósito

El dominio D-SEG gestiona la **función de seguridad pública regional** del GORE, una competencia emergente que integra tres dimensiones:

1. **Gobernanza**: Participación en el Sistema de Seguridad Pública regional
2. **Cartera**: Gestión de proyectos de prevención del delito y asistencia a víctimas
3. **Operaciones**: Centro Integrado de Emergencia y Seguridad (CIES)

### Marco Legal Habilitante

| Norma                                     | Contenido Relevante                                                                                                                                                      |
| ----------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Proyecto de Ley Prevención del Delito** | Crea División de Prevención del Delito en GORE. Funciones: diseñar, ejecutar y evaluar políticas de prevención social, situacional y comunitaria + asistencia a víctimas |
| **Ley 21.730 (Min. Seguridad Pública)**   | Crea Sistema de Seguridad Pública. GORE participa en Consejo Regional de Seguridad. Sistema Nacional de Protección Ciudadana incluye televigilancia                      |
| **Ley 19.628**                            | Protección de la Vida Privada. Rige tratamiento de datos audiovisuales                                                                                                   |
| **ISO 11064**                             | Diseño ergonómico de centros de control                                                                                                                                  |

### M2. Cartera de Proyectos de Seguridad

**Objetivo:** Gestionar el portafolio de inversiones en prevención del delito y asistencia a víctimas.

#### Tipologías de Proyectos

| Tipo                       | Descripción                                     | Ejemplos                                          | Mecanismo Típico         |
| -------------------------- | ----------------------------------------------- | ------------------------------------------------- | ------------------------ |
| **Prevención Situacional** | Infraestructura y equipamiento para disuasión   | Cámaras, iluminación, CPTED, alarmas comunitarias | FNDR, Circular 33        |
| **Prevención Social**      | Programas de intervención en factores de riesgo | Reinserción, prevención juvenil, mediación        | Glosa 06, Transferencias |
| **Prevención Comunitaria** | Fortalecimiento de capacidades locales          | Capacitación, redes vecinales, alertas ciudadanas | Glosa 06, 8% FNDR        |
| **Asistencia a Víctimas**  | Apoyo y acompañamiento a víctimas de delitos    | Centros de atención, redes de derivación          | Glosa 06, Convenios      |
| **Equipamiento FFOO**      | Dotación a Fuerzas de Orden y Seguridad         | Vehículos, equipos, infraestructura policial      | FNDR, Circular 33        |

#### Reglas Especiales (vs. IPR Genéricas)

| Regla                        | Descripción                                                          | Justificación                           |
| ---------------------------- | -------------------------------------------------------------------- | --------------------------------------- |
| **Fast-Track Emergencia**    | Aprobación acelerada (24-48h) para situaciones críticas de seguridad | Art. 4° Ley 21.730 (respuesta oportuna) |
| **Alineamiento Obligatorio** | Todo proyecto debe alinearse con Política Regional de Seguridad      | Coherencia con Política Nacional        |
| **Validación SPD**           | Proyectos de prevención requieren visación técnica de SPD            | Asesoría técnica obligatoria            |
| **Coordinación Municipal**   | Proyectos situacionales requieren convenio con municipio receptor    | Sostenibilidad operativa                |

#### Entidades

```yaml
Proyecto_Seguridad:
  hereda_de: D-FIN.IPR  # Extiende entidad IPR
  atributos_adicionales:
    - tipo_prevencion: ENUM [SITUACIONAL, SOCIAL, COMUNITARIA, ASISTENCIA_VICTIMAS, EQUIPAMIENTO_FFOO]
    - alineamiento_politica: Ref[Politica_Regional_Seguridad]
    - validacion_spd: Boolean
    - convenio_municipal: Ref[D-NORM.Convenio]  # Si aplica
    - indicadores_impacto: Array[Indicador]
  reglas:
    - SI tipo_prevencion IN [SITUACIONAL, EQUIPAMIENTO_FFOO] ENTONCES convenio_municipal REQUERIDO
    - SI monto > 1000 UTM ENTONCES validacion_spd REQUERIDO

Convenio_Seguridad_Municipal:
  hereda_de: D-NORM.Convenio  # Extiende entidad Convenio
  atributos_adicionales:
    - municipio: Ref[D-COORD.Actor]
    - plan_comunal_ref: String  # Referencia al Plan Comunal de Seguridad Pública
    - compromisos_operativos: Array[Compromiso]  # Mantención, operación, personal
    - indicadores_seguimiento: Array[Indicador]
```

### M3. Operaciones CIES

**Objetivo:** Operar el Centro Integrado de Emergencia y Seguridad para vigilancia, coordinación y respuesta.

#### Infraestructura

| Componente            | Especificación                                                                | Referencia                      |
| --------------------- | ----------------------------------------------------------------------------- | ------------------------------- |
| **Sala de Monitoreo** | 77,03 m², diseño ISO 11064, Video Wall 6x2m (4K), 7 estaciones de trabajo     | Manual Sala Monitoreo           |
| **Red de Cámaras**    | 209 puntos en 21 comunas (140 PTZ 4K + 69 multisensor/panorámicas)            | Especificaciones Televigilancia |
| **Conectividad**      | Red híbrida: 80% fibra óptica (100Mbps) + 20% inalámbrica (50Mbps), 316 nodos | Especificaciones Televigilancia |
| **Almacenamiento**    | 60 días a máxima resolución, redundancia RAID, cifrado                        | Manual Sala Monitoreo           |
| **VMS**               | HikCentral, compatibilidad ONVIF, arquitectura federada                       | Especificaciones Televigilancia |

#### Gestión de Evidencia Audiovisual

| Proceso                | Descripción                                               | Plazo                                           | Normativa             |
| ---------------------- | --------------------------------------------------------- | ----------------------------------------------- | --------------------- |
| **Almacenamiento**     | Grabación continua de todas las cámaras                   | 30 días (estándar), 60 días (capacidad técnica) | Ley 19.628            |
| **Cautela Ciudadana**  | Preservación a solicitud de víctima/testigo               | Hasta 6 meses                                   | Manual Sala Monitoreo |
| **Entrega Formal**     | Solo por orden judicial, oficio Fiscalía o instrucción MP | Según requerimiento                             | Código Procesal Penal |
| **Eliminación Segura** | Borrado irreversible (wiping) al cumplir plazo            | Automático                                      | Ley 19.628            |

## 6. Indicadores Clave

| Indicador                         | Descripción                                              | Meta        |
| --------------------------------- | -------------------------------------------------------- | ----------- |
| **Tiempo de Respuesta**           | Tiempo promedio desde detección hasta despacho de unidad | < 3 minutos |
| **Cobertura de Cámaras**          | % de puntos críticos con cobertura de vigilancia         | 100%        |
